# 竞争对手监测系统部署指南

## 国内可访问的部署方案

以下是在国内不需要VPN即可访问的静态网站部署方案：

### 方案一：Gitee Pages（推荐）
Gitee Pages是码云提供的免费静态网站托管服务，国内访问速度快，操作流程与GitHub Pages类似。

#### 部署步骤：
1. **注册并登录Gitee**
   - 访问 https://gitee.com/ 注册账号并登录

2. **创建仓库**
   - 点击右上角"+"号，选择"新建仓库"
   - 填写仓库信息：
     - 仓库名称：建议使用英文（如 `competitor-monitoring`）
     - 仓库介绍：竞争对手监测系统
     - 选择"公开"仓库
     - 点击"创建"

3. **配置本地Git仓库**
   ```bash
   # 设置Gitee用户名和邮箱
   git config --global user.name "你的Gitee用户名"
   git config --global user.email "你的Gitee邮箱"
   
   # 添加Gitee远程仓库
   git remote add gitee https://gitee.com/你的用户名/competitor-monitoring.git
   
   # 推送代码到Gitee
   git push -u gitee main
   ```

4. **启用Gitee Pages**
   - 进入Gitee仓库页面
   - 点击"服务" → "Gitee Pages"
   - 配置选项：
     - 选择分支：`main`
     - 选择目录：`/(根目录)`
   - 点击"启动"
   - 等待部署完成，获取访问URL（如：`https://你的用户名.gitee.io/competitor-monitoring/`）

### 方案二：阿里云OSS静态网站托管
阿里云OSS（对象存储服务）提供静态网站托管功能，访问速度快，可靠性高。

#### 部署步骤：
1. **注册并登录阿里云**
   - 访问 https://www.aliyun.com/ 注册并登录

2. **创建OSS Bucket**
   - 进入OSS控制台
   - 点击"创建Bucket"
   - 配置：
     - 地域：选择离你最近的地域
     - 存储类型：标准存储
     - 读写权限：公共读
   - 点击"确定"

3. **开启静态网站托管**
   - 进入Bucket详情页
   - 点击"基础设置" → "静态网站托管"
   - 开启静态网站托管
   - 设置首页：`index.html`
   - 可选：设置404页面

4. **上传文件**
   - 点击"文件管理" → "上传文件"
   - 批量上传项目所有文件

5. **获取访问域名**
   - 在静态网站托管页面获取访问域名（如：`你的bucket名称.oss-cn-beijing.aliyuncs.com`）

### 方案三：腾讯云COS静态网站托管
腾讯云COS（对象存储）提供静态网站托管服务，国内访问速度快。

#### 部署步骤：
1. **注册并登录腾讯云**
   - 访问 https://cloud.tencent.com/ 注册并登录

2. **创建存储桶**
   - 进入COS控制台
   - 点击"创建存储桶"
   - 配置：
     - 地域：选择离你最近的地域
     - 访问权限：公共读
   - 点击"确定"

3. **开启静态网站**
   - 进入存储桶详情页
   - 点击"基础配置" → "静态网站"
   - 开启静态网站
   - 设置首页：`index.html`

4. **上传文件**
   - 点击"文件列表" → "上传文件"
   - 批量上传项目所有文件

5. **获取访问域名**
   - 在静态网站配置页面获取访问域名

### 方案四：华为云OBS静态网站托管
华为云OBS（对象存储服务）提供静态网站托管功能。

#### 部署步骤：
1. **注册并登录华为云**
   - 访问 https://www.huaweicloud.com/ 注册并登录

2. **创建桶**
   - 进入OBS控制台
   - 点击"创建桶"
   - 配置：
     - 区域：选择离你最近的区域
     - 桶策略：公共读
   - 点击"立即创建"

3. **配置静态网站托管**
   - 进入桶详情页
   - 点击"基础配置" → "静态网站托管"
   - 开启静态网站托管
   - 设置默认首页：`index.html`

4. **上传文件**
   - 点击"对象" → "上传对象"
   - 批量上传项目所有文件

5. **获取访问域名**
   - 在静态网站托管配置页面获取访问域名

## 部署注意事项

1. **域名配置**
   - 所有方案都支持绑定自定义域名
   - 如需绑定域名，需要在DNS解析中添加对应的解析记录

2. **更新部署**
   - Gitee Pages：推送代码后，需要在Gitee Pages页面手动点击"更新"
   - 云服务商OSS/COS/OBS：上传新文件后自动更新

3. **资源路径**
   - 确保项目中所有资源引用使用相对路径（如 `href="styles.css"`）
   - 避免使用绝对路径（如 `href="/styles.css"`）

4. **访问速度**
   - 使用云服务商的CDN加速可以进一步提升访问速度
   - Gitee Pages已内置CDN加速

## 常见问题

### 为什么页面显示404？
- 检查首页文件名是否为 `index.html`
- 检查文件路径是否正确
- 检查存储桶/仓库的权限设置

### 为什么CSS/JS文件加载失败？
- 检查文件引用路径是否正确
- 检查文件是否已成功上传

## 本地测试

在部署前，建议在本地测试：

```bash
python3 -m http.server 8000
```

然后访问 http://localhost:8000 测试网站功能。
